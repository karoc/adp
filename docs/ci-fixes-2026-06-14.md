## Problem 1: Chinese Output Breaking CI Assertions

### Issue
```
runtime-smoke: workspace doctor output missing expected text: ok
runtime-smoke: workspace doctor Git env output missing expected text: warning
```

### Root Cause
Phase 1 enhancement changed doctor output from English to Chinese:
- "ok" → "✓ 工作区 'game-a' 健康"
- "[error]" → "[错误]"
- "[warning]" → "[警告]"

But CI smoke tests had hardcoded English keyword assertions:
```bash
assert_contains "$output" "ok"
assert_contains "$output" "warning"
```

### Solution
Implement **bilingual output** format:
- Status: "✓ 工作区 'game-a' 健康 (ok)"
- Labels: "[错误 error]", "[警告 warning]", "[信息 info]"

This satisfies both:
- Human readers (Chinese primary, English secondary)
- Automated tests (English keywords present)

### Files Changed
- `internal/cli/doctor_renderer.go` - Added English keywords
- `internal/cli/doctor_version_test.go` - Updated test assertions
- `internal/cli/workspace_base_commands_test.go` - Updated test assertions

## Lessons Learned

### 1. Internationalization Requires Dual Strategies

**Challenge**: Support human readers in their native language while maintaining CI compatibility.

**Solution**: Bilingual output with primary language first, CI keywords in parentheses.

**Pattern**:
```
[主要语言 ci-keyword]
健康 (ok)
错误 (error)
```

**Benefit**: Serves both audiences without compromise.

---

### 2. Code Quality Checks Drive Better Architecture

**Challenge**: File grew to 934 lines, exceeding 700-line limit.

**Insight**: The limit is not arbitrary—large files indicate poor separation of concerns.

**Solution**: Split by logical boundaries (diagnostics vs suggestions).

**Takeaway**: Treat quality checks as architecture guidance, not obstacles.

---

### 3. Document Classification Matters

**Challenge**: Not all markdown files are "documentation" in the user sense.

**Discovery**: 
- User docs (install.md, faq.md) → Need bilingual
- Meta docs (audit reports, phase reports) → Project-internal
- Technical docs (design docs) → Developer-focused
- Example internals (prompts, memory) → Agent-consumed

**Solution**: Exemptions based on purpose, not just location.

**Principle**: **Bilingual requirement applies to user-facing content only.**

---

### 4. Test Assertions Should Be Resilient

**Challenge**: Hardcoded text assertions broke when output format evolved.

**Pattern**:
```bash
# Brittle
assert_contains "$output" "workspace game-a is healthy"

# Resilient  
assert_contains "$output" "game-a"
assert_contains "$output" "ok"
```

**Lesson**: Test for semantic keywords, not exact phrasing.

---

### 5. Iterative Debugging is Efficient

**Observation**: Each CI run revealed a new problem.

**Strategy**:
1. Fix one issue
2. Push and wait for CI
3. Analyze next failure
4. Repeat

**Why it works**:
- Avoids fixing imaginary problems
- Real CI environment catches edge cases
- Each fix is independently verifiable
